什么是HTTP

定义

Hyper Text Transfer Protocol（超文本传输协议）是基于TCP/IP的应用层协议。主要是用来规定客户端和服务端之间数据传输的格式。

结构

  • 请求报文

    • 请求行：请求方法（get，post，put，delete），url地址，协议版本

    • 请求头 （键值对形式）

    • 请求体（存放参数）

  • Tip

    URL（Uniform Resource Loactor） 统一资源定位符。

    格式：schema://host:port/path/?queryString

  • 响应报文

    • 响应行： 协议版本，状态码， 状态描述（OK）
    • 响应头：（sever，expires）
    • 响应体： html

  • Tip 状态码

    • 1xx：初始请求已经接受，继续处理客户端发送剩余的请求

    • 200：成功

    • 301：重定向，永久移动，原先url不可用

    • 302：临时重定向，原先url可用

    • 400：请求语法错误

    • 401：要求省份验证

    • 403：拒绝请求

    • 404：资源不存在

    • 5xx：服务器错误。

HTTP调用的过程

首先是应用层的客户端程序发起请求的HTTP数据，请求分别经过传输层，网络层链路层，物理层进行一系列的处理。

  • 应用层：在TCP三次握手后，客户端和服务器同时开发资源空间，通socket进行传输数据。

  • 传输层：TCP通过三次握手建立可靠的连接并且告知网络层

  • 网络层：网络层根据路由表寻址，找到下一跳。

  • 链路层：根据arp表找到其他设备。

数据发送完成后，传输层的TCP进行四次挥手。同时服务端应用程序收到的数据可以根据HTTP的协议进行解析，获取内容。

简单实现HTTP传输

client

import java.io.InputStream;
import java.io.OutputStream;
import java.net.Socket;

public class Client {
//    请求行
    private static StringBuffer requestLine;
//    请求头
    private static StringBuffer requestHeader;
//    请求体
    private static StringBuffer requestBody;

    private static StringBuffer requestContent;

    public static void main(String[] args) throws Exception{
//        发送
        Socket socket = new Socket("127.0.0.1",8888);
        OutputStream outputStream = socket.getOutputStream();
        requestLine.append("GET").append(" ").append("http://www.baidu.com").append(" ").append("HTTP1.1").append("\n");
        requestHeader.append("HOST: www.baidu.com").append(" ").append("\n");;
        requestContent.append(requestLine).append(requestHeader).append(requestContent);
        outputStream.write(requestContent.toString().getBytes());

//        接收
        InputStream inputStream = socket.getInputStream();
        byte[] bytes = new byte[1024];
        int len = inputStream.read(bytes);
        System.out.println(new String(bytes,0,len));
    }
}

server

import java.io.InputStream;
import java.net.ServerSocket;
import java.net.Socket;

public class Server {
//   响应行
    private static StringBuffer responseLine;
//    响应头
    private static StringBuffer responseHeader;
//    响应体
    private static StringBuffer responseBody;
//    返回的html文本
    private static StringBuffer responseHtml;

    private static StringBuffer responseContent;

    public static void main(String[] args) throws Exception{
//        发送
        ServerSocket serverSocket = new ServerSocket(8888);
        Socket socket = serverSocket.accept();
        responseLine.append("HTTP/1.1").append(" ").append(200).append(" ").append("OK").append("\n");
        responseHeader.append("Expires: Sat, 22 Aug 2020 09:11:38 GMT").append(" ").append("Server: BWS/1.1").append("\n");
        responseHtml.append("<h1>Hello World!!<h1>");
        responseContent.append(responseLine).append(responseHeader).append(responseHtml);
        socket.getOutputStream().write(responseContent.toString().getBytes());


//        接收
        InputStream inputStream = socket.getInputStream();
        byte[] bytes = new byte[1024];
        int len = inputStream.read(bytes);
        System.out.println(new String(bytes,0,len));
    }
}

HTTP解决了哪些问题，以及使用场景

  • 解决的问题

    HTTP协议制定了传输数据的统一标准，使得数据传输简单、灵活、易于扩展。

    • 优点

      • 灵活：HTTP允许传输任意类型的数据对象。

      • 无状态：是指协议对于事务处理没有记忆能力。缺少状态意味着如果后续处理需要前面的信息，则它必须重传，在服务器不需要先前信息时它的应答就较快。也可能导致每次连接传送的数据量增大。

      • 无连接：无连接的含义是限制每次连接只处理一个请求。服务器处理完客户的请求，并收到客户的应答后，即断开连接。采用这种方式可以节省传输时间。

  • 使用场景

    • 一般用在c/s模式
    • 如今也用在微服务之间的通讯调用。如feign，HTTPClient

HTTP的缺点

  • 通信使用明文（不加密），内容可能会被窃听。
  • 由于HTTP是基于TCP/IP的，必须经过三次握手，四次挥手的过程。因此会比较慢一些。
  • 对于微服务系统而言，当我们对业务需求的响应速度要求较高的时候，例如高不发项目。HTTP（这里是1.X版本而言）解析是基于文本，因此效率对大大降低。因此很多RPC框架采用自定义的通讯协议。

RPC（拓展）

概念

RPC（Remote Procedure Call）远程过程调用，简单的理解是一个节点请求另一个节点提供的服务。

结构

  • function 客户端的对象函数。
  • stub 代理对象。
  • socket 套接字。

总结

RPC是一种概念模型，只要是实现可远程的调用就是RPC。HTTP的传输调用就是一种RPC。区别就是RPC可以不用HTTP协议，可以自定义传输协议。目的就是为了实现更高效的调用。例如谷歌的gRPC框架。它的协议是采用HTTP2，HTTP2相较于HTTP先前版本采用文本协议的格式解析的优势是采用了二进制格式进行解析。速度更快。同时对hearder进行了压缩。
